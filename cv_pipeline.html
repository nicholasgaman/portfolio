● Computer Vision & AI

Building Number Extraction Pipeline

The objective was to develop a robust computer perception pipeline capable of identifying Curtin bulding numbers and reading the numbers from images around campus. The primary challenge lay in ensuring the system could handle diverse environmental factors, including varying lighting, textures, background and foreground obstructions, and varying perspectives.

● The Approach

Architecture & Design

The task was split into four stages, in order to seperate the different steps of the pipeline:

  • 1. Bulding Number Detection: Utilised transfer learning with the YOLOv8s model which was then trained on nearly 8,000 images to identify building number bounding boxes.
  • 2. Digit Segmentation: Employed the lighter weight YOLOv8n (nano) model to isolate individual digits from the extracted building numbers.
  • 3. Digit Recognition: Built a custom Convolutional Neural Network (CNN) using TensorFlow and Keras to classify the digits.
  • 4. End-to-End Integration: Combined the prior three stages into a continuous pipeline that extracts building numbers from raw images while avoiding negative test cases.

● Optimisation

Technical Problem Solving

In order to train my custom CNN, I generated my own augmented training dataset. Since the building numbers are almost all the same font on campus (Sansa Soft Pro), I was able to use the font to generate a large set of training data by applying augmentations to the images of the numbers. The augmentations applied rotation, perspective transformations, positional transformations, Gaussian noise, contrast and brightness variations, downscaling, random cropping and drawing random lines to mimic background and foreground features around the numbers. To resolve vanishing gradient issues and prevent overfitting, I also implemented batch normalization and dropout layers within the custom CNN.

In order to generate the training data for stages 1 and 2 of the task, a method called pseudo labelling was used. Since I had a large amount of unlabelled data and limited time to manually label thousands of images I used pseudo labelling. This process involved using a smaller manually labelled dataset which was used to train an initial model. This initial model, was then used to label the remaining unlaballed data which was then manually verified to avoid the propogation of errors to the final model. Now that the full dataset was labelled the final model was trained on the expanded dataset. This method allowed me to save a great deal of time while still acquiring excellent results on the final models.
